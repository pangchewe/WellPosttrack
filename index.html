Well Postrack

Advanced Posture Analysis System

Version 2.0 March 2025
1

Executive Summary

Well-Postrack is an advanced system for detecting posture disorders using two image views: Posterior and Lateral. It automatically calculates body angles from key landmarks to generate detailed posture reports, aiding physiotherapists in diagnosis.

Purpose

The system streamlines the diagnostic process by reducing manual tasks like angle measurement and questionnaire responses, allowing physiotherapists to focus on treatment plans.

Benefits

  • Reduces repetitive tasks for physiotherapists
  • Provides standardized posture scores
  • Identifies common posture disorders
  • Generates comprehensive reports
2

System Architecture

The Well Postrack system is separated into two main components: Front-End and Back-End, both deployed as Docker containers on Microsoft Azure.

Front-End

  • Developed with React framework (Vite as builder)
  • UI Library: Chakra-UI
  • Hosted at: well-postrack.azurewebsites.net
  • GitHub Repository: github.com/Invitrace-AI/well-post

Main Functionalities:

  1. Obtain lateral and posterior view images
  2. Collect user questionnaire responses
  3. Annotate body landmarks (manual or auto-generated)
  4. Generate posture analysis reports

Back-End

  • Developed with Flask framework
  • Uses customized keypoint detection model
  • Hosted at: well-postrack-server.azurewebsites.net
  • GitHub Repository: github.com/Invitrace-AI/well-post

Main API Endpoints:

  • /api/predict_keypoints/<orientation>
  • /api/analyze_images

Data Flow

  1. User uploads posterior and lateral view images
  2. Images are processed by the backend model to detect key landmarks
  3. The system calculates angles based on these landmarks
  4. Posture scores are calculated using the PPSS system
  5. Posture disorders are identified based on angle thresholds
  6. A comprehensive report is generated and returned to the user

Postrack Posture Scoring System (PPSS)

The PPSS (Postrack Posture Scoring System) was designed by professional physiotherapists, inspired by REEDCO. It evaluates posture quality by measuring angles between key body landmarks in both lateral and posterior views.

Lateral View: 10 Points Maximum Posterior View: 12 Points Maximum
Good: 2 Points Fair: 1 Point Poor: 0 Points

Trunk <30° 30-40° >40°

Landmark Detection

The system uses a Standard Landmark format to represent body points:

{organName: (x, y, v)}
where:
x: global x-coordinate (range: 0 to image width)
y: global y-coordinate (range: 0 to image height)
v: landmark visibility (range: 0-1)

Key landmarks are detected either automatically by the model or manually adjusted by the user if needed.

Posture Disorder Detection

Well Postrack can diagnose various posture disorders by analyzing the angles between body landmarks. It also identifies muscle weakness and tightness associated with each disorder.

Disorder View Mild Moderate Severe
Lordosis Lateral 10-15° 15-20° >20°
Kyphosis Lateral 10-15° 15-20° >20°
Forward Head Lateral 5-9° 10-15° >15°
Sway Back Lateral 25-29° 29-35° >35°
Scoliosis Posterior <10° 10-20° >20°

Disorder Diagnosis Process

  1. System calculates angles between key landmarks
  2. Angles are compared against disorder-specific thresholds
  3. Severity level is determined based on angle magnitude
  4. Multiple disorders can be identified simultaneously
  5. Associated muscle weakness and tightness is determined

Comprehensive Analysis

Muscle Analysis

The system identifies specific muscle weaknesses and tightness associated with detected posture disorders, based on a professionally developed mapping.

This information helps physiotherapists create targeted treatment plans.

Report Generation

A comprehensive PDF report is generated including:

  • PPSS scores for each body part
  • Detected posture disorders with severity
  • Identified muscle weaknesses
  • Annotated posture images
  • Questionnaire responses

Technical Implementation

Well Postrack is built using modern web technologies and deployed on Microsoft Azure cloud infrastructure.

Deployment Infrastructure:

  • Both front-end and back-end components are containerized using Docker
  • Docker images are stored in Microsoft Azure Container Registry
  • Applications are hosted on Azure App Service
  • Front-end hosted at: well-postrack.azurewebsites.net
  • Back-end hosted at: well-postrack-server.azurewebsites.net

Deployment Process:

  1. Develop code in GitHub repository
  2. Build Docker image
  3. Upload Docker image to Azure Container Registry
  4. Update App Service to use the latest Docker image

Technology Stack

Front-End

  • HTML/CSS
  • Responsive Design

Back-End

  • Flask Framework

Key Technical Features

  • Real-time Feedback: Immediate visualization of detected landmarks
